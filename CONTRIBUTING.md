# 🤝 المساهمة في المشروع

شكراً لك على اهتمامك بالمساهمة في **نظام إدارة نادي كواكب اللياقة**! 

## 📋 الخطوات الأساسية

### 1️⃣ Fork المشروع
انقر على زر "Fork" في أعلى الصفحة لإنشاء نسخة شخصية من المشروع.

### 2️⃣ استنسخ المستودع
```bash
git clone https://github.com/YOUR-USERNAME/gym-management-system.git
cd gym-management-system
```

### 3️⃣ أنشئ فرع جديد
```bash
git checkout -b feature/your-feature-name
```

اختر اسماً وصفياً للفرع:
- `feature/add-new-feature` - لإضافة ميزة جديدة
- `fix/bug-description` - لإصلاح خطأ
- `docs/documentation-update` - لتحديث التوثيق
- `refactor/code-improvement` - لتحسين الكود

### 4️⃣ اجعل التغييرات
قم بتعديل الملفات والتحسينات المطلوبة.

### 5️⃣ Commit التغييرات
```bash
git add .
git commit -m "وصف واضح للتغييرات"
```

**نصائح لكتابة رسائل Commit جيدة:**
- استخدم اللغة العربية أو الإنجليزية
- كن واضحاً وموجزاً
- ابدأ برفعة (prefix) مناسبة:
  - `✨ feat:` للميزات الجديدة
  - `🐛 fix:` لإصلاح الأخطاء
  - `📝 docs:` لتحديث التوثيق
  - `♻️ refactor:` لتحسين الكود

مثال:
```bash
git commit -m "✨ feat: إضافة ميزة تصفية متقدمة للإيرادات"
```

### 6️⃣ Push التغييرات
```bash
git push origin feature/your-feature-name
```

### 7️⃣ افتح Pull Request
1. اذهب إلى مستودعك على GitHub
2. انقر على "New Pull Request"
3. اختر فرعك والفرع الرئيسي `main`
4. اكتب عنواناً وصفياً للـ PR
5. اشرح التغييرات التي أضفتها
6. انقر "Create Pull Request"

## 📝 إرشادات الكود

### أسلوب الكود (Code Style)
- استخدم **ES6+** والميزات الحديثة
- استخدم **PascalCase** لأسماء المكونات
- استخدم **camelCase** لأسماء المتغيرات
- اكتب **Comments واضحة** باللغة العربية أو الإنجليزية

### مثال على مكون جيد:
```javascript
// مكون بطاقة الإحصائيات
function StatCard({ title, value, icon, color }) {
  // Map للألوان المدعومة
  const colorMap = {
    blue: 'bg-blue-100 border-blue-300',
    red: 'bg-red-100 border-red-300',
    green: 'bg-green-100 border-green-300',
    purple: 'bg-purple-100 border-purple-300'
  };

  return (
    <div className={`${colorMap[color]} border-2 rounded-lg p-6 text-right`}>
      {/* محتوى البطاقة */}
    </div>
  );
}
```

### معايير الأداء
- استخدم `useCallback` و `useMemo` عند الحاجة
- تجنب الـ re-renders غير الضرورية
- قسّم المكونات الكبيرة إلى مكونات صغيرة

### الاختبار
- اختبر كودك قبل الإرسال
- تأكد من أن جميع الميزات تعمل كما هو متوقع
- تحقق من أن الكود لا يكسر الميزات الموجودة

## 🐛 الإبلاغ عن الأخطاء

إذا وجدت خطأ، قم بالإبلاغ عنه عبر [Issues](../../issues):

### نموذج الإبلاغ:
```
العنوان: وصف مختصر للمشكلة

الوصف:
- خطوات تكرار المشكلة
- السلوك المتوقع
- السلوك الفعلي

لقطة شاشة (إن أمكن):
- إرفق لقطة شاشة للمشكلة

البيئة:
- نسخة Node.js:
- نسخة المتصفح:
- نظام التشغيل:
```

## 💡 الاقتراحات والميزات الجديدة

هل لديك فكرة لميزة جديدة؟ 

1. تحقق من [Issues](../../issues) للتأكد من عدم اقتراحها من قبل
2. افتح Issue جديد مع الوسم `enhancement`
3. صف الميزة بوضوح وشرح فوائدها
4. انتظر التعليقات والآراء

## 📚 معايير الجودة

قبل إرسال PR، تأكد من:

- ✅ الكود نظيف وسهل القراءة
- ✅ لا توجد أخطاء في وحدة التحكم (Console)
- ✅ التطبيق يعمل بدون مشاكل
- ✅ جميع الميزات المرتبطة تعمل
- ✅ التغييرات موثقة بشكل جيد
- ✅ تم اتباع معايير الكود
